## 1. AI与大模型基础

### AI Agent（AI智能体）
**是什么**：像一个"虚拟人物"，能主动思考、对话、执行任务的AI程序。
**在这个项目里**：我们要创建巴菲特和芒格的虚拟形象，你能跟他们聊天，他们会用自己的风格回答问题。
**简单理解**：就像电影里的贾维斯（钢铁侠的助手），但这里我们模拟的是投资大师。

### LLM（Large Language Model，大语言模型）
**是什么**：一个读过海量文本的AI，能理解和生成人类语言。
**在这个项目里**：LLM是核心"大脑"，负责理解你的问题并生成巴菲特/芒格风格的回答。
**常见例子**：ChatGPT（GPT-4）、Claude、Llama等。
**简单理解**：像一个非常博学但需要指导的助手，本身没有个性，需要我们"教"它怎么扮演巴菲特。

### Prompt Engineering（提示词工程）
**是什么**：设计给AI的指令，让它按我们要求的方式工作。
**在这个项目里**：我们需要写详细的指令告诉AI："你是巴菲特，你说话喜欢用比喻，你会这样开头..."，这样AI才能演得像。
**简单理解**：就像演员的剧本，告诉AI怎么演好巴菲特这个角色。

### Context（上下文）
**是什么**：AI在对话时记住的之前聊过的内容。
**在这个项目里**：AI需要记住你之前问过什么，才能连贯地对话，不会失忆。
**简单理解**：就像对话的记忆，AI记得前几句说了什么。

### Token（词元）
**是什么**：AI处理文本的基本单位，可以是一个词、一个字或一个标点。
**在这个项目里**：AI处理的内容长度有限制，用token数量计算。
**简单理解**：像计算文字数量，但AI有自己的计算方式。1000个token大约是750个英文单词。

## 2. RAG系统相关

### RAG（Retrieval-Augmented Generation，检索增强生成）
**是什么**：一种让AI"翻书"回答问题的技术。
**在这个项目里**：当AI回答问题时，先从巴菲特/芒格的真实文章、书信、演讲中查找相关内容，然后基于这些真实内容回答。
**为什么重要**：保证回答真实可靠，能追溯来源，而不是AI瞎编。
**简单理解**：像考试时开卷答题，AI先翻资料，再基于资料回答问题。

### Vector Store（向量存储）
**是什么**：一种特殊的数据库，能理解"语义相似性"，而不只是关键词匹配。
**在这个项目里**：当你问"什么是价值投资"，它能找到相关段落，即使这些段落没有完全相同的词。
**简单理解**：像一个超级智能的搜索引擎，能理解你问的意思，而不只是字面文字。

### Embedding（文本嵌入）
**是什么**：把文字转换成数字列表，让AI能理解文字的含义。
**在这个项目里**：把巴菲特的所有话语转换成数字，这样AI能计算哪些段落跟你的问题最相关。
**简单理解**：把文字翻译成AI能理解的"坐标"，相似的文字在"地图"上离得近。

### Semantic Retrieval（语义检索）
**是什么**：根据含义而不是关键词来搜索。
**在这个项目里**：问"怎么选股票"，能找到关于"投资策略"的内容，即使原文没说"选股票"这三个字。
**简单理解**：像聪明的人能理解你问的意思，而不是只会找完全一样的词。

### BM25
**是什么**：一种传统但有效的关键词搜索算法。
**在这个项目里**：作为第二层搜索，补充语义检索，确保不会遗漏直接匹配的内容。
**简单理解**：像百度/谷歌的搜索，找你输入的关键词。

### Citation（引用）
**是什么**：标注回答内容的来源。
**在这个项目里**：AI回答时会告诉你"这句话来自2024年股东信"或"来自《穷查理宝典》第5章"。
**简单理解**：像写论文时标注出处，让内容可信可查。

## 3. 数据处理与存储

### Metadata（元数据）
**是什么**：关于数据的数据，描述信息的标签。
**在这个项目里**：给每段巴菲特的话打上标签，比如：
- 来源：股东信
- 年份：2024
- 主题：投资策略
- 角色：巴菲特

**简单理解**：像给文件贴标签，方便分类和查找。

### Text Splitting（文本分割）
**是什么**：把长文档切分成小段，方便AI处理。
**在这个项目里**：把一本《巴菲特传》切成很多小段，每段有适当的长度，这样AI能更好地理解。
**简单理解**：像把一本书拆成很多张小卡片，每张卡片内容完整，方便AI阅读。

### ChromaDB
**是什么**：一个专门存向量数据的数据库。
**在这个项目里**：存储巴菲特和芒格所有话语的"数字坐标"，支持快速搜索。
**简单理解**：像一个专门的文件柜，专门放"AI能理解的文件"。

## 4. Web开发与接口

### API（Application Programming Interface，应用程序接口）
**是什么**：让不同程序互相"对话"的方式。
**在这个项目里**：
- 我们写的后端程序通过API跟AI模型对话
- 网页前端通过API跟后端对话
- 图像生成工具通过API调用AI画图

**简单理解**：像餐厅服务员，连接厨房（后端）和顾客（前端），传递订单和结果。

### FastAPI
**是什么**：一个用Python写后端服务的框架。
**在这个项目里**：构建Web服务器，处理用户请求，调用RAG系统和AI模型。
**简单理解**：像搭建一个"网站后台"，负责接收请求、处理数据、返回结果。

### Gradio
**是什么**：快速创建网页界面的Python工具。
**在这个项目里**：创建聊天界面，让用户在浏览器里跟巴菲特/芒格对话。
**简单理解**：像快速搭建一个聊天窗口的"积木"，不需要写HTML/CSS/JavaScript。

### Endpoint（端点）
**是什么**：API里的每个"功能入口"。
**在这个项目里**：
- `/chat` - 发送聊天消息的入口
- `/get-voice` - 获取语音的入口
- `/get-avatar` - 获取头像的入口

**简单理解**：像餐厅菜单上的不同菜品，每个端点提供一种服务。

## 5. 图像与语音处理

### Midjourney / DALL-E / Bing Image Creator
**是什么**：AI绘画工具，用文字描述生成图片。
**在这个项目里**：生成巴菲特和芒格的美漫风格头像。
**简单理解**：像给一个很厉害的画师描述你想要的画面，它帮你画出来。

### TTS（Text-to-Speech，文字转语音）
**是什么**：把文字转成人声的AI技术。
**在这个项目里**：把AI生成的巴菲特/芒格的回答变成语音播放出来。
**简单理解**：像给AI装上嘴巴，能把文字念出来。

### STT（Speech-to-Text，语音转文字）
**是什么**：把人说话的声音转成文字。
**在这个项目里**：如果你用语音问问题，把你的语音转成文字，AI才能理解。
**简单理解**：像速记员，把你说的话记录成文字。

### Whisper
**是什么**：OpenAI开源的语音转文字模型。
**在这个项目里**：
- 把股东大会视频转成文字
- 把采访视频转成文字
- 作为训练数据

**简单理解**：一个免费的、非常准确的"听写工具"。

## 6. 部署与平台

### Hugging Face Spaces
**是什么**：一个免费托管AI应用的网站。
**在这个项目里**：把我们的应用部署上去，让全世界都能访问。
**简单理解**：像免费的服务器，专门放AI应用，一键发布。

### GitHub
**是什么**：代码托管平台，程序员用它管理代码。
**在这个项目里**：存储所有项目代码，方便协作和分享。
**简单理解**：像Google Drive，但专门放代码，能记录每次修改历史。

### Deployment（部署）
**是什么**：把开发好的软件放到服务器上，让用户能访问。
**在这个项目里**：把写好的代码放到Hugging Face或服务器，你就能通过网址访问了。
**简单理解**：像开餐厅，装修好了（开发完），要正式营业（部署），让客人来吃饭。

### Local vs Cloud（本地 vs 云端）
**是什么**：
- 本地：在自己电脑上运行
- 云端：在远程服务器上运行

**在这个项目里**：
- 开发阶段：本地运行，方便调试
- 正式使用：云端部署，让大家都能用

**简单理解**：
- 本地：在自己家厨房做饭
- 云端：开餐厅，让所有人来吃

## 7. 开发工具与库

### Python
**是什么**：一种编程语言，AI领域最流行。
**在这个项目里**：用Python写所有代码，因为它有最多的AI工具。
**简单理解**：像一种"通用语言"，AI领域大家都用这个交流。

### Library（库）
**是什么**：别人写好的代码工具包，直接拿来用。
**在这个项目里**：
- llama-index：RAG系统工具包
- FastAPI：Web服务器工具包
- Gradio：网页界面工具包

**简单理解**：像现成的乐高积木，直接拿来拼装，不用从头做。

### Requirements.txt
**是什么**：列出项目需要用到的所有工具包。
**在这个项目里**：确保其他人能一键安装所有依赖。
**简单理解**：像做菜前的"食材清单"，列出需要买哪些菜。

### Environment Variable（环境变量）
**是什么**：存储敏感信息的方式，不写在代码里。
**在这个项目里**：存储API密钥、密码等，不用直接写在代码里，更安全。
**简单理解**：像把密码记在秘密本子上，不写在便利贴上。

### .env File
**是什么**：存环境变量的文件。
**在这个项目里**：.env文件里存各种密钥，代码读取这个文件。
**简单理解**：像密码本，统一管理所有敏感信息。

## 8. 项目特定术语

### 数据来源类型

#### Shareholder Letter（股东信）
- **是什么**：巴菲特每年写给伯克希尔公司股东的信
- **为什么重要**：包含巴菲特最核心的投资思想，是必看材料
- **简单理解**：巴菲特每年的"年终总结"，分享投资心得

#### Annual Meeting（股东大会）
- **是什么**：伯克希尔每年在奥马哈举行的股东大会
- **为什么重要**：巴菲特和芒格现场回答股东问题，是最直接展现他们风格的场合
- **简单理解**：年度"粉丝见面会"，股东可以随便问问题

### 还原度相关概念

#### Style Consistency（风格一致性）
- **是什么**：AI每次说话的风格保持一致
- **在这个项目里**：巴菲特要像巴菲特，不能一会儿像巴菲特一会儿像别人
- **简单理解**：像一个演员，始终不"出戏"

#### Metaphor System（比喻系统）
- **是什么**：巴菲特喜欢用的特定比喻
- **经典例子**：
  - 护城河/城堡：企业竞争优势
  - 雪球：复利效应
  - 糖果店：优秀企业
  - 下金蛋的鹅：赚钱机器

**简单理解**：巴菲特喜欢用这些比喻，AI也要学会用。

#### First Principles（第一性原理）
- **是什么**：从最基本的事实出发思考问题
- **在这个项目里**：巴菲特的思维方式，不被表象迷惑
- **简单理解**：不看花里胡哨的东西，看最根本的事实

#### Circle of Competence（能力圈）
- **是什么**：自己真正理解的范围
- **在这个项目里**：巴菲特和芒格最重要的原则之一，承认自己不懂什么
- **简单理解**：知道什么懂，更重要的，知道什么不懂

## 9. 技术流程术语

### Data Pipeline（数据处理流水线）
**是什么**：数据从原始状态到可用状态的整个流程。
**在这个项目里**：
1. 收集书籍PDF、视频等原始材料
2. 提取文字内容
3. 清洗和格式化
4. 切分成小段
5. 转换成向量
6. 存入数据库

**简单理解**：像工厂的生产线，原材料经过多道工序变成成品。

### Inference（推理）
**是什么**：AI模型实际运行生成答案的过程。
**在这个项目里**：当你问问题，AI模型"思考"并生成回答。
**简单理解**：像演员真正上台表演，把排练的内容演出来。

### Latency（延迟）
**是什么**：从发送请求到收到回应的时间。
**在这个项目里**：你问问题后，AI多长时间能回答。
**简单理解**：像等消息回复的时间，越快体验越好。

## 10. 常见文件类型

### PDF（Portable Document Format）
**是什么**：电子书、文档的常用格式。
**在这个项目里**：《穷查理宝典》、《巴菲特传》等书籍大多是PDF格式。
**简单理解**：像电脑版的"打印文件"，格式固定，不容易乱。

### Markdown（.md）
**是什么**：一种简单的文档格式，用简单符号表示格式。
**在这个项目里**：项目文档、说明文件用Markdown写，容易编辑和阅读。
**简单理解**：像简化版的Word，用`**文字**`表示加粗，用`#`表示标题。

### JSON（JavaScript Object Notation）
**是什么**：一种数据交换格式，易读易写。
**在这个项目里**：存储元数据、配置文件等。
**简单理解**：像结构化的数据表，用花括号和冒号组织数据。

## 总结：项目技术栈（最简化版）

### 核心组件

## 常见问题

### Q: 为什么不用ChatGPT？
**A**: ChatGPT要花钱，且数据不透明。我们用开源/免费模型，零成本开发。

### Q: 本地运行和云端部署有什么区别？
**A**:
- 本地：在你自己电脑上，只有你能用，速度快
- 云端：在服务器上，所有人都能用，但需要联网

### Q: 为什么要做RAG，不能直接让AI扮演？
**A**: RAG保证每个回答都有真实依据，不是AI瞎编。这是"还原度"的核心。

### Q: 数据收集为什么这么重要？
**A**: 数据越全，AI越能理解巴菲特和芒格的真实风格，还原度越高。
